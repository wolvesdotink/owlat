Trigger automations with custom events for trial lifecycle, feature adoption, and more.

## How it works

When you send an event via the API, Owlat checks if any active automation has a matching trigger. If it does, the automation runs for that contact — sending emails, waiting before the next step, or branching on a condition. You define the automations in the dashboard; your code just fires the events.

## Plan your events

Define a consistent naming scheme before you start sending. Here's an example for a SaaS trial lifecycle:

| Event name | When to fire | Suggested properties |
| --- | --- | --- |
| `trial_started` | User creates account | `planName`, `trialDays` |
| `feature_activated` | User tries a key feature | `featureName` |
| `trial_expiring` | 3 days before trial ends | `daysLeft`, `planName` |
| `trial_expired` | Trial period ends | `planName` |
| `plan_upgraded` | User converts to paid | `planName`, `mrr` |

::callout{type="tip" title="Naming conventions"}
Use `snake_case` for event names. Names must start with a letter and can contain letters, numbers, underscores, and hyphens (max 100 characters).
::

## Send events

### Trial started

::code-group
```typescript [TypeScript]
import { Owlat } from '@owlat/sdk-js'

const owlat = new Owlat('lm_live_...')

await owlat.events.send({
  email: 'mira@acme.io',
  eventName: 'trial_started',
  eventProperties: {
    planName: 'Pro',
    trialDays: 14,
  },
})
```

```sh [curl]
curl -X POST https://your-deployment.convex.site/api/v1/events \
  -H "Authorization: Bearer lm_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "email": "mira@acme.io",
    "eventName": "trial_started",
    "eventProperties": {
      "planName": "Pro",
      "trialDays": 14
    }
  }'
```
::

### Feature activated

```typescript
await owlat.events.send({
  email: 'mira@acme.io',
  eventName: 'feature_activated',
  eventProperties: {
    featureName: 'team_invite',
  },
})
```

### Trial expiring

```typescript
await owlat.events.send({
  email: 'mira@acme.io',
  eventName: 'trial_expiring',
  eventProperties: {
    daysLeft: 3,
    planName: 'Pro',
  },
})
```

## Auto-create contacts

If you fire events for users who might not exist as contacts yet, set `createContactIfNotExists`:

```typescript
await owlat.events.send({
  email: 'new-lead@example.com',
  eventName: 'trial_started',
  eventProperties: { planName: 'Pro', trialDays: 14 },
  createContactIfNotExists: true,
})
```

The response tells you whether a contact was created:

```json
{
  "data": {
    "eventId": "evt_m6e...",
    "contactId": "j57...",
    "eventName": "trial_started",
    "triggeredAutomations": 1,
    "contactCreated": true
  }
}
```

## Setting up the automation

Once your code sends events, create the automation in the Owlat dashboard:

1. Go to **Automations → New Automation**
2. Set the trigger to **Event Received** and enter the event name (e.g. `trial_expiring`)
3. Add steps — send an email, wait, send another, etc.
4. Publish the automation

Automations are matched by event **name** only — the event name you set on the trigger must equal the `eventName` you send. Event properties are recorded with the event but do not affect matching, and they are **not** injected into the email templates the automation sends. Automation emails personalize only with contact fields — `{{firstName}}`, `{{lastName}}`, and `{{email}}`.
