title: "Media Library"
description: "Manage, organize, search, and reuse images and files across your emails from one centralized hub."
---

The media library is a centralized hub for managing images and files used across your emails. Instead of uploading images individually each time, you can organize, search, and reuse assets from one place.

## Navigating to the media library

Open **Mail > Media** from the dashboard sidebar. The media library shows all uploaded assets with thumbnails, file names, and metadata.

## Uploading files

Add files to the library in two ways:

- **Drag and drop** — drag one or more files directly onto the media library page
- **Upload button** — click **Upload** and select files from your computer

The media library upload accepts common image types (JPEG, PNG, GIF, WebP, SVG), PDF, and video and audio files. Additional document types (Word, Excel, CSV, and similar) are only attachable via the [Transactional API](/api/transactional), not the dashboard upload picker.

## Searching and filtering

Use the search bar to find assets by file name, alt text, or tags. As your library grows, tags become especially useful for organizing assets by campaign, product, or content type.

## Tags

Tags let you categorize assets for quick filtering:

- Add tags from the asset detail view
- Filter the library by a tag (one tag at a time)
- Use consistent naming conventions (e.g., `product-shots`, `2026-spring`, `logos`)

## Editing metadata

Click any asset to open its detail view. From there you can edit:

- **Alt text** — descriptive text for accessibility and email clients that block images
- **Tags** — categorize the asset for easier discovery

::callout{type="tip"}
Always set alt text on images. It improves accessibility for screen reader users and provides context when images are blocked by email clients.
::

## Using media in the email editor

When you add an image block in the [Email Editor](/guide/email-editor), you can pick from the media library instead of uploading a new file. This keeps your images consistent and avoids duplicates.

1. Add an `/image` block in the editor.
2. Click the image/browse button next to the **Image URL** field in the image settings.
3. Search or browse the media library.
4. Select the image — it's inserted into your email automatically.

## Transactional email attachments

Files from the media library can be attached to [Transactional Emails](/guide/transactional). In the transactional email editor, the attachment panel includes a **Media library** button so you can pick a stored file instead of re-uploading it.

When sending through the [Transactional API](/api/transactional), attachments are supplied inline on the `attachments` field as base64 content or an HTTPS URL — they are not referenced by media-library ID.

## Bulk operations

Select multiple assets to perform a bulk action:

- **Delete** — remove selected assets

## Usage tracking and storage

The media library header shows a summary line such as "42 files · 12 MB used" — the number of files in your library and the total storage they consume.

This helps you keep an eye on storage as your library grows.

::callout{type="info" title="Per-asset usage"}
Open any asset and its detail panel shows a single combined count of the emails (templates, transactional emails, and saved blocks) that currently reference it — so you can tell at a glance whether an image is safe to delete.
::
