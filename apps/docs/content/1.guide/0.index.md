# Guide

Owlat is a modular email platform. Each capability — marketing campaigns, a personal mailbox, a team inbox, AI assistance — is an independent feature you can turn on or off. Pick the parts you need; ignore the rest.

::callout{type="info" title="Toggling features"}
Every feature listed below as **(optional)** is controlled by a feature flag. Toggle them from **Settings → Features** or with `owlat-setup feature <key> <on|off>`. See [Feature flags](/guide/feature-flags) for an overview and [the developer reference](/developer/feature-flags) for the underlying mechanics.
::

## Personal Email (Postbox)

Per-user mailboxes with a webmail UI and native IMAP/SMTP for desktop clients. Each user gets their own inbox, folders, filters, aliases, and signatures — independent of the marketing/campaign side of Owlat.

- [Postbox](/guide/postbox) — set up your personal mailbox, compose, filter, and connect native clients _(optional · flag: `postbox`)_
- [Connect external mailbox](/guide/postbox#connect-an-external-mailbox) _(optional · flag: `mail.external`)_ — connect an existing Gmail / Fastmail / company mailbox over IMAP+SMTP, no sending domain required
- [Migrate from Google](/guide/migrate-from-google) — import your full Gmail history into Owlat
- [Desktop App](/guide/desktop-app) — the native desktop client for Postbox and the team inbox

## Team Inbox

Shared, threaded inbox for a team to triage inbound mail together. Pairs with the AI agent to auto-classify mail and suggest replies.

- [Team Inbox](/guide/team-inbox) — shared threading and triage _(optional · flag: `inbox`)_
- [Chat](/guide/chat) — real-time chat surface alongside the inbox _(optional · flag: `chat`)_
- [Code Tasks](/guide/code-tasks) — surface bug reports from inbound mail as developer tasks _(optional · flag: `inbox.codeTasks`, requires `inbox` + `ai.agent`)_

Other inbound channels (SMS, WhatsApp) can be configured and receive messages — see [Communication Channels](/guide/channels).

## AI

AI features are off by default and require an AI provider. Pick a hosted backend with an API key (OpenAI, Anthropic, Google, Azure OpenAI, OpenRouter) or a model you host yourself (Ollama, vLLM, llama.cpp) from **Settings → AI Provider** — every AI feature uses it, with no per-feature wiring.

- [Connect your AI](/guide/connect-your-ai) — choose the language backend and embeddings; swap backends with one settings edit
- Master AI toggle _(flag: `ai`)_ — required by every AI feature
- [AI Agent & Autonomy](/guide/ai-agent) — classify inbound mail, draft replies, and (optionally) auto-send high-confidence replies _(flags: `ai.agent`, `ai.autonomy`)_
- [Knowledge Graph](/guide/knowledge-graph) — typed memory entries and relations the agent draws on for context _(flag: `ai.knowledge`)_
- [Files](/guide/files) — upload documents the agent can search and cite
- [AI Assistant](/guide/ai-assistant) — multi-turn, streaming, tool-calling assistant and `@assistant` replies in team chat _(flag: `ai.assistant`)_
- [AI dashboards](/guide/ai-agent#visualization-agent-adaptive-dashboards) _(flag: `ai.visualizations`)_ — generate charts from prompts

## Security & Deliverability

- [Deliverability](/guide/deliverability) — monitor inbox placement and reputation
- [Sending Security & Scanning](/guide/security-scanning) — content, attachment, and URL scanning before mail leaves
- Content scanning _(flag: `scan.content`)_ — block spam/phishing pre-send (on by default)
- File scanning _(flag: `scan.files`)_ — ClamAV malware scan on attachments (on by default)
- URL reputation _(optional · flag: `scan.urls`)_ — Safe Browsing checks on links
- Domain verification _(flag: `domains.verification`)_ — SPF/DKIM/DMARC checks (on by default)
- DKIM auto-rotation _(flag: `domains.dkimRotation`)_ — scheduled signing key rotation (on by default)
