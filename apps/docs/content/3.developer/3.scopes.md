Each part of the monorepo has a clear scope. Apps are deployable services, packages are shared libraries consumed by one or more apps.

## Apps

| App | Stack | Scope |
| --- | --- | --- |
| `apps/web` | Nuxt 4 + Vue 3 | Main user dashboard ‚Äî email builder, campaigns, audience management, automations, settings |
| `apps/api` | Convex | Serverless backend ‚Äî database, authentication (BetterAuth), real-time subscriptions, REST API, cron jobs |
| `apps/mta` | Hono + GroupMQ | Custom Mail Transfer Agent ‚Äî direct SMTP delivery, DKIM signing, IP warming, bounce processing, rate limiting |
| `apps/imap` | Node + IMAP | IMAP server (port 993) ‚Äî translates IMAP4rev1 commands into Convex queries against per-user mailboxes |
| `apps/mail-sync` | Node worker | Syncs users' external IMAP/SMTP accounts ‚Äî inbound mail in near-real-time via IMAP IDLE, relays outbound sends |
| `apps/code-worker` | Node worker | Polls Convex for queued code tasks and executes them (GitHub task runner) |
| `apps/desktop` | Tauri 2 + Rust | Desktop shell ‚Äî bundles the `apps/web` SPA, connects to one or more deployments (multi-workspace) |
| `apps/setup-cli` | TypeScript (Bun) | Self-host setup CLI ‚Äî the `owlat quickstart` installer that deploys a full instance |
| `apps/updater` | Node | Self-host update worker ‚Äî HTTP control surface that pulls and applies Docker Compose updates |
| `apps/docs` | Nuxt + Content | Developer documentation ‚Äî guides, API reference, architecture decisions |
| `apps/marketing` | Nuxt | Public marketing site ‚Äî landing pages, pricing, changelog |

## Packages

| Package | Stack | Scope |
| --- | --- | --- |
| `packages/shared` | TypeScript | Shared types and data ‚Äî block definitions, editor types, email client compatibility matrix |
| `packages/email-renderer` | TypeScript | Headless email HTML engine ‚Äî table-based layouts, VML for Outlook, CSS inlining, dark mode, AMP format |
| `packages/email-builder` | Vue 3 | Notion-like email editor ‚Äî document canvas, block previews, settings popover, slash commands |
| `packages/email-previewer` | Vue 3 | Preview and analysis ‚Äî cross-client rendering simulation, compatibility scoring, email diffs |
| `packages/email-scanner` | TypeScript | Email security scanning ‚Äî spam/phishing keyword detection, file magic-byte validation, ClamAV client, URL reputation (Safe Browsing) |
| `packages/channels` | TypeScript | Multi-channel adapters ‚Äî email, SMS, WhatsApp, generic webhook, native chat + inbound channel registry |
| `packages/ui` | Nuxt Layer | Shared UI primitives ‚Äî toast notifications, error boundaries, auto-imported composables |
| `packages/plugin-kit` | TypeScript | Public plugin contracts (npm) ‚Äî `definePlugin`, capability constants, every contribution interface |
| `packages/plugin-host` | TypeScript | Private plugin enforcement kernel ‚Äî capability/flag checks, ordering, gate results, untrusted-text policy; shared by Convex and Nuxt |
| `packages/plugin-codegen` | TypeScript | Build tool owning the single bundled-plugin composition point and the generated catalogs/registries |
| `packages/plugin-cli` | TypeScript (Bun) | `owlat-plugins` ‚Äî create/add/remove/codegen/dev over `plugins.config.ts` |
| `packages/sdk-js` | TypeScript | JavaScript SDK (npm) ‚Äî contacts, transactional sends, events, topics |
| `packages/sdk-java` | Java | Java SDK (Maven) ‚Äî same API surface as sdk-js for JVM consumers |

## Who imports what

- **web** imports `email-builder`, `email-previewer`, `email-renderer`, `ui`, `shared`, and the generated Convex client `@owlat/api` (real-time queries & mutations)
- **api** imports `email-renderer`, `email-scanner`, `channels`, and `shared`
- **docs** and **marketing** import `ui`
- **imap**, **mail-sync**, and **code-worker** are standalone Node workers ‚Äî they talk to `api` over the Convex client
- **mta** is standalone ‚Äî it receives HTTP requests from `api` and delivers mail directly
- **sdk-js** and **sdk-java** are standalone ‚Äî they talk to the public REST API exposed by `api`
- **plugin-kit** is imported by plugin packages, `plugin-host`, `plugin-codegen`, `api`, `web`, and `code-worker`; **plugin-host** is imported by `api` and `web`; **plugin-codegen** runs only at build time. Bundled plugin packages themselves are reachable **only** from generated composition files ‚Äî see the [Plugin Platform](/developer/plugin-platform) chapter
