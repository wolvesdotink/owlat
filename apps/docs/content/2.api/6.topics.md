Manage topic membership through authenticated endpoints.

## Authentication

These endpoints require an API key with the `topics:write` scope, passed as a bearer token:

```http
Authorization: Bearer <key>
```

Requests without a valid key return `401`; requests with a key that lacks the `topics:write` scope return `403`.

## Endpoints

| Method | Path                                          | Description               |
| ------ | --------------------------------------------- | ------------------------- |
| `POST` | `/api/v1/topics/:topicId/contacts`            | Add contact to topic      |
| `DELETE` | `/api/v1/topics/:topicId/contacts/:emailOrId` | Remove contact from topic |

## Add contact to topic

```http
POST /api/v1/topics/:topicId/contacts
```

Body accepts either `email` or `contactId`:

```json
{
  "email": "jane@example.com"
}
```

or

```json
{
  "contactId": "j57..."
}
```

Success response (`201 Created`):

```json
{
  "data": {
    "success": true,
    "contactId": "j57...",
    "topicId": "k19...",
    "doiStatus": "pending"
  }
}
```

`doiStatus` can be:

- `not_required`
- `pending`
- `confirmed`

## Remove contact from topic

```http
DELETE /api/v1/topics/:topicId/contacts/:emailOrId
```

`emailOrId` can be URL-encoded email or contact ID.

Success response (`200 OK`):

```json
{
  "data": {
    "success": true,
    "removed": true
  }
}
```
